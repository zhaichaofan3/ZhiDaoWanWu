# 部署到 Sealos 集群

## 步骤 1：准备工作

1. 确保你已经在 Sealos 上创建了 MySQL 数据库服务，并且数据库服务正在运行。
2. 确保你已经获取了数据库的连接信息，包括：
   - 内网地址：`second-hand-mysql.ns-fxwy6sdx.svc`
   - 端口：`3306`
   - 用户名：`root`
   - 密码：`665nsv25`
   - 数据库名：`second_hand`

## 步骤 2：创建 Docker 镜像

### 方法 1：使用 Sealos 内置的构建功能

1. 登录 Sealos 控制台
2. 点击左侧导航栏的 "应用管理" -> "新建应用"
3. 选择 "从代码仓库部署"
4. 输入你的代码仓库地址（如果你的代码在 GitHub、GitLab 等代码托管平台上）
5. 选择合适的构建配置，Sealos 会自动构建 Docker 镜像

### 方法 2：本地构建 Docker 镜像

1. 在本地项目目录中执行以下命令构建 Docker 镜像：
   ```bash
   docker build -t your-image-name .
   ```
2. 将构建好的镜像推送到 Docker Hub 或其他镜像仓库：
   ```bash
   docker push your-image-name
   ```
3. 在 Sealos 上创建应用时，使用你推送到镜像仓库的镜像地址

## 步骤 3：创建应用

1. 登录 Sealos 控制台
2. 点击左侧导航栏的 "应用管理" -> "新建应用"
3. 填写应用基本信息：
   - 应用名称：例如 `second-hand-app`
   - 镜像：使用方法 1 或方法 2 中的镜像
   - 容器数量：根据需要设置，建议至少 1 个
   - 端口：设置为 `4000`（与应用代码中的端口一致）
4. 点击 "高级配置"，设置环境变量（如果需要）
5. 点击 "部署" 按钮，开始部署应用

## 步骤 4：配置网络

1. 部署完成后，点击应用名称进入应用详情页
2. 点击 "网络配置" 标签页
3. 配置域名：
   - 可以使用 Sealos 提供的默认域名
   - 也可以绑定自己的域名
4. 点击 "保存" 按钮，完成网络配置

## 步骤 5：访问应用

1. 部署完成后，在应用详情页中可以看到应用的访问地址
2. 点击访问地址，即可打开应用
3. 测试应用功能，确保一切正常

## 步骤 6：配置数据库连接

应用代码中已经配置了数据库连接信息，使用的是 Sealos 集群内部的服务名称：

```javascript
const dbConfig = {
  host: 'second-hand-mysql.ns-fxwy6sdx.svc',
  port: 3306,
  user: 'root',
  password: '665nsv25',
  database: 'second_hand',
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
};
```

当应用部署到 Sealos 集群中时，会自动解析 `second-hand-mysql.ns-fxwy6sdx.svc` 这个服务名称，连接到你的 MySQL 数据库。

## 故障排除

### 应用无法连接到数据库

1. 检查数据库服务是否正在运行
2. 检查数据库连接配置是否正确
3. 检查应用是否与数据库在同一个集群中
4. 检查网络策略是否允许应用访问数据库

### 应用启动失败

1. 查看应用日志，了解具体错误信息
2. 检查依赖是否安装正确
3. 检查端口是否被占用
4. 检查环境变量是否设置正确

### 前端页面无法访问

1. 检查前端构建是否成功
2. 检查后端服务是否正在运行
3. 检查前端是否正确配置了后端 API 地址
4. 检查网络连接是否正常

## 总结

通过以上步骤，你可以将应用部署到 Sealos 集群中，并使用 Sealos 上的 MySQL 数据库服务。如果遇到问题，可以查看应用日志，了解具体错误信息，然后进行相应的调整。
